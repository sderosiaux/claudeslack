How It Works

Send messages from Slack, Claude Code executes on your machine

Why Slack?

Leverage a familiar interface with powerful capabilities

Works Offline

Queue messages while commuting. Responses arrive when you're back online. Async input AND output.

Voice Input

Slack's built-in microphone means hands-free coding prompts. Dictate while walking.

Threads = Sessions

Each thread is a task with full context continuity. New message = new task. Organized by default.

Shareable

Invite teammates to project channels. Collaborate on tasks in real-time.

File Uploads

Drop images or code files. Saved to your project folder, analyzed by Claude instantly.

Free Tier

Works with Slack's free plan. No additional costs beyond Claude Code subscription.

Why Not Just SSH?

Fair question. Here's the honest comparison.

ssh + tmux + claude

  • No extra dependencies
  • Direct terminal access
  • Full control over session
  • Requires network/VPN to home
  • Clunky on phone keyboards
  • No async - need connection

claudeslack

  • Works offline (queue messages)
  • Voice input via Slack
  • Threads = persistent sessions
  • Polished mobile UX (swipe channels/threads)
  • Adds Slack as dependency
  • Data through Slack servers
  • Another daemon to maintain

Use SSH if you have Tailscale/VPN and don't mind terminal on mobile.
Use claudeslack if you want async messaging, voice input, or hate phone keyboards.

Quick Start

Up and running in 3 steps

~15 min first time, ~5 min if you've set up Slack apps before
1

Create a Slack App

Go to api.slack.com/apps and create a new app with Socket Mode enabled. See the setup guide for required scopes.

2

Install & Configure

Clone the repo and run setup with your tokens.

git clone https://github.com/sderosiaux/claudeslack.git
cd claudeslack
go build -o claudeslack .
./claudeslack setup xoxb-YOUR-BOT-TOKEN xapp-YOUR-APP-TOKEN
3

Start Listening

Run the listener and start coding from Slack!

./claudeslack listen --projects-dir ~/code/projects

Security & Threat Model

Be honest about what this is

What Actually Happens to Your Data

Your prompt
Slack servers → Your machine
File contents Claude reads
Your machine → Anthropic API
Claude's response
Anthropic API → Your machine → Slack servers
File writes
Local only

Translation: Your code files stay local, but their contents go to Anthropic when Claude reads them. Your prompts and responses are stored in Slack's infrastructure.

Safeguards

  • Allowlist of Slack user IDs
  • Config file with 0600 permissions
  • Socket Mode (no public webhook)
  • Open source - audit the code

What This Is

  • A weekend hack, not enterprise software
  • Convenience tool for personal use
  • Uses --dangerously-skip-permissions
  • Single-user, single-machine design

Don't Use For

  • Production codebases
  • Code under NDA
  • Client work with confidentiality clauses
  • Shared workspaces without consent
